《数据结构(C语言版)(第3版)—揭安全》

《数据结构(C语言版)(第3版)—揭安全》

概论

基本概念：

  1. 数据结构学科历史定位：数据结构研究计算结果不是纯数学的程序中的运算对象、对象之间的关系、对对象的操作，近五六十年其作为计算机科学从离散数学中
  2. 逻辑结构术语
    1. 数据由数据元素组成
    2. 数据元素由数据项组成
    3. 可归于同一个集合（贴上同一个标签）的数据元素们统称数据对象
  3. 数据结构：本书认为数据结构是相互之间存在一种或多种特定关系的数据元素的集合。实际上这个词被人用烂了，不同的含义太多
  4. 四类基本结构：集合，线树图
    1. 集合：对集合而言，数据元素之间是孤立的，没有半毛钱关系。
    2. 线性结构：数据元素之间存在一对一关系。
    3. 树形结构：数据元素之间存在一对多关系。
    4. 图状结构：又网状结构，数据元素之间存在多对多的关系【我个人认为，树是特殊图】。
  5. 我个人认为，一元信息（只依赖一个元素即可完整表示信息）不需要借助任何东西，只要在程序中约定【譬如虚数约定虚部数字为i所在即可】，二元信息在顺序存储可存储器的物理上的肩并肩关系来表示，再往上的多元信息需要借助高级语言表示，由此观之整个数据结构也不过是一个的多元信息。
  6. 存储结构方面的名词：
    1. 映像：又物理结构、存储结构。数据结构在计算机中的表示。
    2. 元素：又结点。数据元素概念在存储结构上的对应“映像”。
    3. 数据域：数据项概念在存储结构上的对应“映像”。
    4. 顺序映像、非顺序映像：又顺序存储结构和非顺序存储结构。区别是借不借助存储器物理上的连续来表达元素的逻辑结构。
    5. 虚拟存储结构：借助具体的高级语言去表达存储结构，那么元素和元素之间关系的信息转移到高级语言中去，叫虚拟存储结构。所以我们大部分时间只是学怎么写虚拟存储结构。

数据类型和抽象数据类型：

  1. 数据结构的符号表示：数据结构名=<D,S>。其中C是数据元素的集合，R是数据关系的集合，尖括号表序偶关系。
  2. 抽象数据类型：数据结构+操作。
  3. 抽象数据类型分三种，原子类型【自身没由更低数据类型来定义】， 固定聚合类型【数据元素的数量固定不变】，可变聚合类型：【数据元素的数量可变】。其中后两种统称结构类型。另有多形数据类型【元素的成分可以不同】。

算法和算法分析：

  1. 算法五大重要特性：算法的步骤有效有穷而确定，算法本身可以没有输入但不能没有输出。
  2. 算法质量标准：正确【语法正确=>特定输入正确=>刁难数据正确=>一切合法输入都正确】，可读【人类能读懂】，健壮【能应对非法输入】，时空消耗。
  3. 两种方法计算程序执行时间：事后统计，事先预估。
    1. 事后统计：依靠计算机内部计时器毫秒级计算，缺点是你得真运行一遍而且会受计算机软硬件环境影响，有时误差不可接受。
    2. 事先预估的方法：我们只在纯数学上比较算法优劣，所以只估算问题规模（原操作）对算法的函数关系，问题规模（原操作）用变量n表示。
  4. 事先预估法小总结：我们需要知道的是纯数学意义上的时间复杂度，逻辑如下：纯数学时间复杂度函数T(n)≈拟合时间复杂度函数O(f(n))∝问题规模f(n)∝基本操作总重复次数n。
    如果f(n)的解析式是一个常，一次，二次，对数，指数函数，对应时间复杂度函数称常量，线性，平方，对数，指数阶，在特定定义域之内用什么阶算法的时间复杂度较低，可在如下函数图像中比较。

  1. 事先预估法的特殊情况处理：如果算法的时间复杂度会随输入数据的不同而不同【例如起泡法对最佳输入程序甚至不需要起泡】，处理方法要么计算平均时间复杂度【所有可能输入对应时间复杂度的期望】，要么以最坏结果为准。如果算法的时间复杂度必须考虑多种基本操作，可以用赋权的方式处理。
  2. 空间复杂度：继承时间复杂度中f(n)的定义，效仿T(n)定义一个空间复杂度函数S(n)。
    数据分为程序（指令+常量+变量）和输入，如果输入量和算法无关就只考虑临时变量，如果输入量受算法影响就同时考虑输入本身的空间占用，如果对不同的输入临时变量都是常数，算法称原地工作。

第2章 线性表及其顺序存储

线性表：

  1. 线性表这三个字就是线树图的那个线，其可以被顺序或链式存储【多年难题】。

顺序表：

  1. 顺序表就是顺序存储的线性表。

  2. 写法备注：因为不同语言写法上的通用问题，我只利用比较通用的特性来描述虚拟存储结构。ADT中所有方法的首尾都是头部检查元素的合法性【一般都是表空了表满了之类的】，尾部返回需要的值。如果需要改动就输入指针，只是运算就输入表列就行。除了计数器这种需要int的之外，数据元素的数据类型并不硬性规定，只会写一个typedef然后再宏定义上去根据需要替换元素类型。需要创建时就规划死的那种存储长度也不硬性规定，也是写一个typedef然后宏定义。所有的位置记录，根据需要有本、前、后三种，实际中根据需要要调整这些位置，也就是加减一。

  3. 顺序存储结构：

    1. 头文件元素：数组，有效位置。
    2. 方法：
      1. 置空表/初始化。
      2. 尾部插入。
      3. 打印整表。
      4. 判断空表。
      5. 查找值x的位置。
      6. 取得第i结点值。
      7. 在p位置插入值x。
      8. 删除第p位置。

栈：

  1. 后进先出【FILO,first in last out】的顺序表。

  2. 栈顶插删，栈底不动。

  3. 顺序存储结构：

    1. 头文件元素：数组，栈顶位置。
    2. 方法：
      1. 初始化。
      2. 判断空栈。
      3. 取栈顶值。
      4. 栈顶插入。
      5. 栈顶删除。

队列：

  1. 先进先出【FIFO，first in first out】的顺序表。

  2. 队尾插，队首删。

  3. 顺序存储结构：

    1. 头文件元素：数组，队首位置，队尾位置。
    2. 方法：
      1. 初始化。
      2. 判断空队列。
      3. 打印队列。
      4. 取队首值。
      5. 队尾插入。
      6. 队首删除。

  4. 因为插删会令队首位置不断前进，导致数组可用长度越来越短，设计上令队列首尾相接，这种思想的队列称为循环队列。

  5. 循环队列的空满条件都是队首位置等于队尾位置，解决循环队列的空满判断问题有两种先人想到的方法，其一是判断“队首位置等于队尾位置”是由队首删除一位导致【队满】还是队尾插入一位导致【队空】，其二是减少一个存储位，这样队首位置等于队尾位置+1就判断队满，队首位置等于队尾位置时队空。

  6. 顺序循环队列方法：

    1. 队尾插入。
    2. 队首删除。

第3章 线性表的链式存储

链式存储：

  1. 链表写法上的问题主要是空间的分配和回收，也就是C语言的malloc和free函数，对不同高级语言需要注意解决一下这个问题。
  2. 链式结构和顺序结构的一大区别是不再整体申请空间，头文件描述的是节点而不是整体。

单链表：

  1. 链式存储结构：

    1. 头文件元素：数据值，存储后一个结点地址的节点型指变。
    2. 方法
      1. 创建单链表。
      2. 打印单链表。
      3. 取第i结点的地址。
      4. 在第i结点后插入值x。
      5. 删除值为x的结点。

带头结点的单链表：

  1. 单链表由于各种条件判断写起来很复杂，需要增加点什么来辅助判断，所以特别设置第一个节点为头结点，此头结点head不属于单链表，只辅助写方法。换句话说，带头结点的单链表只是单链表的另一种写法而已，区别是设置一个头结点写起来会比较方便。
  2. 链式存储结构：
    1. 头文件元素：头文件和单链表是一样的，不表。
    2. 方法：
      1. 创建带头结点的单链表。
      2. 打印带头结点的单链表。
      3. 取第i节点的地址。
      4. 在第i节点后插入值x。
      5. 删除值为x的结点。

循环单链表：

  1. 类似循环队列的设计，这样一直走下去就能再次访问某一结点。方法是令最后一个结点的下一个结点地址从NULL改为第一个结点的位置。
  2. 链式存储结构：
    1. 头文件元素：和单链表一样。
    2. 方法：
      1. 创建循环单链表。
      2. 取最后结点的地址。
      3. 输出循环单链表。
      4. 查找值为x的结点。
      5. 在第i结点后插入值x。
      6. 删除值为x的结点。

双链表：

  1. 双链表就是单链表增加一个指向前驱结点的数据元素。
  2. 链式存储结构：
    1. 头文件元素：数据值，存储后一个结点地址的结点型指变，存储前一个结点地址的结点型指变。
      typedef int datatype;
      typedef struct link_node{
      datatype info; //数据值
      struct link_node *llink，*rlink; //存储前一个和后一个结点地址的结点型指变
      }node;
    2. 方法：
      1. 创建双链表。
      2. 打印双链表。
      3. 取第i结点的地址。
      4. 在第i结点插入值x。
      5. 删除值为x的结点。

链式栈：

  1. 链式存储结构：
    1. 头文件元素：和单链表是一样的。
    2. 方法：
